Previous Meeting Discussion

Additional sample programs

We went over program ideas to add to the sample programs folder.

We already have implementations for the Eight Queens Problem solver and a Binary Search Tree.

Due to the 8-bit program counter limitation, programs are limited to 256 instructions, so writing programs in anything above assembly is not feasible yet, except for something like BASIC perhaps.

However, in the previous meeting we discussed implementing the following:

  • Sorting algorithms

      • DFS recursive implementation

New Business

16-bit address bus

Ideas were brought up regarding how to implement a 16-bit address bus:

One way is to add another address register for the high byte of the address, called a page register, and to have a separate instruction for updating the page register.
In this way, we could maintain the simplicity of needing only an 8-bit immediate or a single register to update the current RAM address.
However, this would likely require extra control signals (since we write to the page register and address register separately) as well as use up an extra opcode or two for additional instructions to update the page register.

Another approach is to make the address register 16-bits wide and to require load / store instructions to specify the entire address (this likely would mean using both an immediate and a register value to construct the address).
The advantage of this is that we can likely just revamp current load/store instructions to accomodate this approach, but the downside is that we need to use up a register for every load / store.

Miscellaneous

Comments and concerns

The point was brought up that since the ByteFrost is still in its initial stage and the only code that was written so far was written by its creators, the ISA can be modified without respect to backwards compatibility.
The goal is to have a generic and powerful RISC ISA that is intuitive to use and doesn't pose a bottleneck for developers.
